Types & Variabelen

 ·  De bouwstenen van Java

Een variabele is een benoemde plek in het geheugen waar je een waarde bewaart. In Java geef je elke variabele een vast type, zodat de compiler weet hoeveel ruimte er nodig is en welke bewerkingen mogen. Je hebt primitieve types zoals int, double en boolean, en objecttypes zoals String. Het type kies je eenmaal bij het declareren en daarna mag de variabele enkel waarden van dat type krijgen. Zo vang je veel fouten al op tijdens het compileren in plaats van pas bij het uitvoeren.

Een variabele is een plek in het geheugen met een naam waar je een waarde in bewaart. Voor je ze gebruikt geef je aan welk type ze heeft, want dat bepaalt wat erin mag. Java kent primitieve types zoals int voor gehele getallen, double voor kommagetallen en boolean voor waar of niet waar. Daarnaast bestaan objecttypes zoals String voor tekst. Je declareert een variabele met haar type en geeft ze daarna een waarde met =.

If / Else

 ·  Beslissingen in Java

Met een if-structuur laat je je programma keuzes maken. Java evalueert de voorwaarde tussen de haakjes tot true of false en voert enkel het bijbehorende blok uit. Met else vang je het geval op waarin de voorwaarde niet klopt, en met else if keten je meerdere voorwaarden na elkaar. De voorwaarden worden van boven naar beneden gecontroleerd en bij de eerste die klopt stopt de keten. Zo stuur je de uitvoering afhankelijk van de toestand van je gegevens.

Met een if-structuur laat je je programma kiezen. Je zet een voorwaarde tussen haakjes, en alleen als die waar is loopt het blok eronder. Met else vang je het tegenovergestelde geval op, en met else if test je meerdere mogelijkheden na elkaar. Voorwaarden bouw je met vergelijkingen zoals groter dan of gelijk aan. Je combineert ze met en (&&) en of (||), zodat je code zelf bepaalt welke weg hij neemt.

    System.out.println("Uitstekend");

    System.out.println("Goed");

    System.out.println("Voldoende");

    System.out.println("Onvoldoende");

    System.out.println("Lekker buiten!");
}

if (temp < 5 || !zonaig) {
    System.out.println("Beter binnen blijven");
}
⚠️Voor Strings: gebruik nooit == in een if-statement. Gebruik naam.equals("Bram") of naam.equalsIgnoreCase("jan").

Switch

 ·  Meerdere keuzes overzichtelijk

Een switch kiest één tak op basis van de waarde van één variabele. Het is een overzichtelijk alternatief voor een lange reeks if-else als je veel vaste mogelijkheden vergelijkt. Java springt naar de case die overeenkomt en voert die uit tot het een break tegenkomt. Vergeet je de break, dan loopt de uitvoering door naar de volgende case. Met default vang je alle waarden op die in geen enkele case voorkomen.

Een switch is een nettere manier om één waarde met veel vaste mogelijkheden te vergelijken. Je geeft de variabele mee, en elke case beschrijft één waarde waarop je reageert. Na de code van een case zet je break, anders valt de uitvoering door naar de volgende case. Het default-blok vangt alles op wat bij geen enkele case past. Voor een lange reeks gelijkheidstests leest een switch vaak duidelijker dan een hele rij if-else.

Wanneer gebruiken?

Een switch is handig als je één variabele vergelijkt met meerdere vaste waarden. Overzichtelijker dan een lange if/else-if keten.

Klassieke switch

int dag = 3;

switch (dag) {
    case 1:
        System.out.println("Maandag");
        break;
    case 2:
        System.out.println("Dinsdag");
        break;
    case 3:
        System.out.println("Woensdag");
        break;
    default:
        System.out.println("Onbekende dag");
        break;
}

Switch met String

String kleur = "rood";

switch (kleur) {
    case "rood":
        System.out.println("Stop!");
        break;
    case "oranje":
        System.out.println("Opgelet!");
        break;
    case "groen":
        System.out.println("Rijd!");
        break;
    default:
        System.out.println("Onbekende kleur");
}

Meerdere cases tegelijk

int maand = 4;

switch (maand) {
    case 12:
    case 1:
    case 2:
        System.out.println("Winter");
        break;
    case 3:
    case 4:
    case 5:
        System.out.println("Lente");
        break;
    default:
        System.out.println("Zomer of herfst");
}
⚠️Vergeet break; niet! Zonder break valt Java door naar de volgende case (fall-through).

While-lus

 ·  Herhalen zolang een voorwaarde geldt

Een while-lus herhaalt een blok code zolang een voorwaarde waar blijft. Java controleert die voorwaarde telkens vóór een nieuwe ronde, dus als ze meteen false is loopt de lus nul keer. Binnen de lus moet je iets veranderen dat de voorwaarde ooit false maakt, anders blijf je oneindig draaien. Je gebruikt while vooral als je vooraf niet weet hoeveel herhalingen er nodig zijn. Een typisch voorbeeld is invoer blijven vragen tot de gebruiker een geldig antwoord geeft.

Een while-lus herhaalt een blok zolang een voorwaarde waar blijft. Java test die voorwaarde eerst, dus het kan dat de lus nul keer loopt. Bij een do-while draai je het om, daar loopt het blok altijd minstens één keer omdat de test pas achteraan komt. Zorg dat er telkens iets in de lus verandert, anders blijf je oneindig herhalen. While gebruik je vooral als je niet op voorhand weet hoe vaak je moet herhalen.

While

De lus wordt uitgevoerd zolang de voorwaarde true is. Als de voorwaarde al meteen false is, wordt de lus nooit uitgevoerd.

int teller = 0;

while (teller < 5) {
    System.out.println("teller = " + teller);
    teller++;   // NIET vergeten, anders oneindige lus!
}
// Uitvoer: teller = 0, 1, 2, 3, 4

Do-while

De lus wordt minstens één keer uitgevoerd, daarna pas gecontroleerd.

int getal = 10;

do {
    System.out.println("getal = " + getal);
    getal--;
} while (getal > 0);

Break en continue

int i = 0;
while (i < 10) {
    i++;
    if (i == 3) continue; // sla 3 over, ga naar volgende iteratie
    if (i == 7) break;    // stop de lus volledig bij 7
    System.out.println(i);  // print 1, 2, 4, 5, 6
}
⚠️Vergeet de teller nooit te verhogen in een while-lus, anders krijg je een oneindige lus en bevriest je programma!

For-lus

 ·  Herhalen met een teller

Een for-lus is ideaal als je op voorhand weet hoe vaak je iets wil herhalen. In één regel zet je de teller op, geef je de stopvoorwaarde en bepaal je de stap die na elke ronde gebeurt. Java voert die drie delen in een vaste volgorde uit, zodat de teller netjes meeloopt. Je gebruikt een for-lus vaak om via een index over een reeks of array te lopen. Het houdt de luslogica compact en op één plaats.

Een for-lus gebruik je als je wel weet hoe vaak je iets moet herhalen. In de kop staan drie delen, namelijk de startwaarde van de teller, de voorwaarde om door te gaan, en de stap die na elke ronde gebeurt. Zolang de voorwaarde klopt voert Java de inhoud uit en past daarna de teller aan. Zo loop je bijvoorbeeld netjes van 0 tot een bepaald getal. Voor het overlopen van een array of lijst is dit de standaardkeuze.

Standaard for-lus

Bestaat uit drie delen: start, voorwaarde en stap.

// for (start; voorwaarde; stap)
for (int i = 0; i < 5; i++) {
    System.out.println("i = " + i);
}
// Uitvoer: i = 0, 1, 2, 3, 4

Achterwaarts tellen

for (int i = 10; i >= 0; i--) {
    System.out.println(i);
}
// 10, 9, 8 ... 1, 0

For-each lus (enhanced for)

Voor het doorlopen van arrays of lijsten, zonder index nodig:

String[] namen = {"Bram", "Stijn", "Noor"};

for (String naam : namen) {
    System.out.println(naam);
}

Geneste for-lussen

// Tafel van vermenigvuldiging
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        System.out.print(i * j + "\t");
    }
    System.out.println();
}
// 1  2  3
// 2  4  6
// 3  6  9
✅Gebruik een for-lus als je op voorhand weet hoeveel keer je wilt herhalen. Gebruik een while-lus als dat pas tijdens de uitvoering bekend is.

Methoden

 ·  Herbruikbare blokken code

Een methode is een stuk code met een naam dat je apart kan oproepen, zodat je het niet hoeft te herhalen. Je geeft eventueel parameters mee als invoer, en met return geef je een resultaat terug. Het returntype vooraan bepaalt wat er terugkomt, of void als er niets teruggegeven wordt. Door taken in methoden op te splitsen wordt je code leesbaarder en makkelijker te testen. Je roept een methode op met haar naam, gevolgd door de argumenten tussen haakjes.

Een methode is een stukje code met een naam dat je apart zet en opnieuw kunt oproepen. Zo vermijd je dat je dezelfde code blijft herhalen. Tussen de haakjes geef je parameters mee, dat zijn de waarden waarmee de methode werkt. Het returntype zegt wat de methode teruggeeft, en met void geef je aan dat ze niets teruggeeft. Door je programma in methoden te knippen blijft alles overzichtelijk en makkelijk te testen.

Opbouw van een methode

// toegang  returntype  naam       (parameters)
public static int     somVan      (int a, int b) {
    return a + b;
}

Methode zonder returnwaarde (void)

public static void drukAf(String tekst) {
    System.out.println(tekst);
}

// Oproepen:
drukAf("Hallo!");

Methode met returnwaarde

public static double berekenOppervlakte(double breedte, double hoogte) {
    return breedte * hoogte;
}

// Oproepen en opslaan:
double opp = berekenOppervlakte(5.0, 3.0); // opp = 15.0

Methode met boolean return

public static boolean isEven(int getal) {
    return getal % 2 == 0;
}

if (isEven(4)) {
    System.out.println("4 is even");
}

Overloading

Meerdere methodes met dezelfde naam, maar verschillende parameters:

public static int som(int a, int b) {
    return a + b;
}
public static int som(int a, int b, int c) {
    return a + b + c;
}
public static double som(double a, double b) {
    return a + b;
}
📌Een methode met static kan je oproepen zonder eerst een object te maken. In een klasse (OOP) laat je static weg.

Enumeratie (enum)

 ·  Vaste lijst van waarden

Een enum is een eigen type met een vaste, beperkte lijst van mogelijke waarden, zoals de dagen van de week of een status. Je gebruikt het wanneer een variabele maar een handvol geldige opties mag hebben. De compiler controleert dan dat je enkel bestaande waarden gebruikt, wat typfouten met losse strings voorkomt. Elke enumwaarde is een vast object dat je via de typenaam aanspreekt, bijvoorbeeld Dag.MAANDAG. Je kan een enum ook als attribuut van een klasse of als returntype van een methode gebruiken.

Een enum is een eigen type met een vaste, beperkte lijst van mogelijke waarden. Denk aan de dagen van de week of de seizoenen, waar maar een handvol opties bestaan. Je schrijft de waarden meestal in hoofdletters en gebruikt ze via de naam van de enum. Het voordeel is dat de compiler fouten tegenhoudt, want een verkeerde waarde bestaat gewoon niet. Zo wordt je code leesbaarder en veiliger dan met losse tekst of getallen.

Wat is een enum?

Een enum is een speciaal type dat een vaste reeks benoemde waarden bevat. Ideaal voor categorieën zoals kleuren, seizoenen, statussen, enz.

Enum definiëren

public enum Seizoen {
    LENTE, ZOMER, HERFST, WINTER
}

Enum gebruiken

Seizoen huidig = Seizoen.ZOMER;

if (huidig == Seizoen.ZOMER) {
    System.out.println("Het is zomer!");
}

Enum in een switch

switch (huidig) {
    case LENTE:
        System.out.println("Bloemen bloeien");
        break;
    case ZOMER:
        System.out.println("Warm en zonnig");
        break;
    case HERFST:
        System.out.println("Bladeren vallen");
        break;
    case WINTER:
        System.out.println("Koud en donker");
        break;
}

Enum met extra info

public enum Dag {
    MAANDAG(true), DINSDAG(true), WOENSDAG(true),
    DONDERDAG(true), VRIJDAG(true),
    ZATERDAG(false), ZONDAG(false);

    private boolean werkdag;

    Dag(boolean werkdag) {
        this.werkdag = werkdag;
    }

    public boolean isWerkdag() { return werkdag; }
}

// Gebruik:
Dag vandaag = Dag.VRIJDAG;
System.out.println(vandaag.isWerkdag()); // true

Alle waarden doorlopen

for (Seizoen s : Seizoen.values()) {
    System.out.println(s);
}
// LENTE, ZOMER, HERFST, WINTER
✅Enum-waarden vergelijk je met == (niet .equals()). Dat is één van de weinige uitzonderingen in Java.

Arrays

 ·  Geordende reeks van waarden

Een array bewaart meerdere waarden van hetzelfde type in één variabele, netjes achter elkaar in het geheugen. De lengte leg je vast bij het aanmaken en die kan daarna niet meer veranderen. Je bereikt elk element via zijn index, die start bij 0 en loopt tot de lengte min 1. Met een for-lus loop je vlot over alle elementen. Arrays zijn snel, maar gebruik een ArrayList als het aantal elementen moet kunnen groeien of krimpen.

Een array is een rij vakjes van hetzelfde type met een vaste lengte. Je maakt ze aan met een grootte, en daarna bereik je elk vakje via zijn index. Let op, het tellen begint bij 0, dus het eerste element zit op index 0 en het laatste op de lengte min 1. Met de eigenschap length vraag je op hoeveel plaatsen er zijn. Een array is snel, maar de grootte ligt vast zodra je hem hebt aangemaakt.

Array aanmaken

// Methode 1: direct initialiseren
int[] scores = {85, 92, 78, 95, 61};

// Methode 2: lengte opgeven, later waarden toewijzen
int[] scores = new int[5];
scores[0] = 85;
scores[1] = 92;

// String-array
String[] namen = new String[3];
namen[0] = "Bram";

Array uitlezen

int[] getallen = {10, 20, 30, 40, 50};

System.out.println(getallen[0]);       // 10 — index start bij 0!
System.out.println(getallen.length);   // 5 — aantal elementen

Array doorlopen met for

int[] getallen = {10, 20, 30, 40, 50};

// Klassieke for-lus
for (int i = 0; i < getallen.length; i++) {
    System.out.println(getallen[i]);
}

// For-each (korter)
for (int g : getallen) {
    System.out.println(g);
}

Nuttige bewerkingen

int[] getallen = {10, 20, 30, 40, 50};

// Som berekenen
int som = 0;
for (int g : getallen) { som += g; }  // som = 150

// Maximum zoeken
int max = getallen[0];
for (int g : getallen) {
    if (g > max) max = g;
}
// max = 50

// Array sorteren
Arrays.sort(getallen); // {10, 20, 30, 40, 50} → gesorteerd
⚠️Een array heeft een vaste lengte. Als je dynamisch elementen wil toevoegen of verwijderen, gebruik dan een ArrayList!

OOP & Klassen

 ·  Basis van objectgeoriënteerd programmeren

Objectgeoriënteerd programmeren draait rond klassen en objecten. Een klasse is de blauwdruk die beschrijft welke gegevens (attributen) en welk gedrag (methoden) iets heeft; een object is een concrete invulling daarvan. Attributen maak je meestal private en je stelt ze in of vraagt ze op via getters en setters, wat we encapsulatie noemen. De constructor vult een nieuw object met beginwaarden, en toString geeft er een leesbare voorstelling van. Zo bundel je samenhorende gegevens en logica in één geheel.

Objectgericht programmeren draait rond klassen en objecten. Een klasse is de blauwdruk die beschrijft welke gegevens iets heeft en wat het kan doen. Een object is een concreet exemplaar dat je van die klasse maakt met new. De gegevens noem je attributen, en je houdt ze meestal private zodat ze enkel via methoden bereikbaar zijn. Met getters en setters lees of wijzig je die waarden gecontroleerd, en met een toString geef je een leesbare voorstelling van het object.

Wat is een klasse?

Een klasse is een sjabloon/blauwdruk voor objecten. Ze bevat attributen (eigenschappen) en methodes (gedrag).

Structuur van een klasse

public class Toetsenbord {
    // 1. Attributen (private!)
    private String serienummer;
    private String merk;
    private boolean draadloos;
    private BigDecimal prijs;

    // 2. Constructor(s)
    public Toetsenbord(String merk) {
        this.merk = merk;
    }
    public Toetsenbord() {     // default constructor
        this.merk = "Logitech";
        this.draadloos = true;
    }

    // 3. Getters & Setters
    public String getMerk() { return merk; }
    public void setMerk(String merk) { this.merk = merk; }
    public boolean isDraadloos() { return draadloos; }

    // 4. Andere methodes
    public BigDecimal berekenPrijs(int aantal) {
        return prijs.multiply(BigDecimal.valueOf(aantal));
    }

    // 5. toString
    @Override
    public String toString() {
        String tekst = "";
        tekst += "Merk: " + merk + "\n";
        tekst += "Draadloos: " + draadloos + "\n";
        return tekst;
    }
}

Meerdere methodes met dezelfde naam maar verschillende parameters:

Abstracte Klassen & Overerving

 ·  abstract · extends · super · @Override

Met overerving laat je een subklasse de attributen en methoden van een superklasse hergebruiken via extends. Een abstracte klasse dient enkel als basis: je kan er geen objecten van maken, maar wel gemeenschappelijke code in plaatsen. Een abstracte methode legt enkel de vorm vast en moet door elke subklasse worden ingevuld. In de constructor van de subklasse roep je eerst super(...) op om de geërfde velden te vullen. Met @Override herschrijf je een methode, en met super.toString() hergebruik je de versie van de superklasse.

Overerving laat een klasse voortbouwen op een andere. De subklasse erft alle attributen en methoden van de superklasse en voegt er eigen dingen aan toe. Een abstracte klasse is een superklasse waarvan je zelf geen objecten maakt, ze dient enkel als basis. Daarin kun je abstracte methoden zetten zonder inhoud, die elke subklasse dan verplicht invult. Met super roep je de constructor of een methode van de superklasse op, zodat je geërfde code hergebruikt in plaats van te herhalen.

Wat is een abstracte klasse?

Een abstracte klasse is een klasse waarvan je geen objecten kunt aanmaken. Ze dient als basis (superklasse) voor subklassen.

public abstract class Klassenaam {
    // ... inhoud
}
⚠️new Klassenaam() is NIET mogelijk bij een abstracte klasse.

Abstracte methode

Een abstracte methode heeft geen implementatie. Elke subklasse is verplicht ze in te vullen.

public abstract returntype methodenaam(parameterlijst);

Superklasse met protected attributen

protected attributen zijn zichtbaar in de klasse zelf en in elke subklasse, zodat een subklasse rechtstreeks aan de geërfde velden kan. Een abstracte methode mag een enum teruggeven.

public enum Brandstof {
    BENZINE,
    DIESEL,
    ELEKTRISCH
}
import java.math.BigDecimal;

public abstract class Voertuig {
    protected String merk;            // protected: ook zichtbaar in subklassen
    protected BigDecimal prijs;
    protected double gewichtInKg;

    public Voertuig(String merk, BigDecimal prijs, double gewichtInKg) {
        this.merk = merk;
        this.prijs = prijs;
        this.gewichtInKg = gewichtInKg;
    }

    public String getMerk() {
        return merk;
    }

    public BigDecimal getPrijs() {
        return prijs;

BigDecimal

BigDecimal gebruik je voor geldbedragen en andere getallen waar afrondingsfouten niet mogen, want double rekent niet exact met kommagetallen. JDBC (Java Database Connectivity) is de standaardmanier om vanuit Java met een databank te praten. JDBC: Prepared Statements

 ·  geprecompileerde SQL met ? als placeholder

Een prepared statement is een SQL-instructie waarin je waarden vervangt door een vraagteken als placeholder. Je maakt het met prepareStatement en vult daarna elke placeholder in met setString, setInt, setDouble enzovoort, geteld vanaf 1. De databank compileert de instructie één keer, wat herhaald gebruik sneller maakt. Omdat de waarden los van de SQL worden meegegeven, beschermt dit ook tegen SQL-injectie. Uitvoeren doe je met executeQuery voor een SELECT of executeUpdate voor een wijziging.

Een prepared statement is een geprecompileerde SQL-instructie. Voordelen: makkelijk te parametriseren en sneller door de precompilatie. Je zet een ? als placeholder voor elke parameter.

Prepared statement voor een SQL-query (SELECT)

Je maakt het aan met connectie.prepareStatement(...), stelt de parameters in met setDouble, setString, ... en voert uit met executeQuery().

import java.sql.Connection;

JDBC: Transacties

 ·  commit · rollback · autocommit

Een transactie groepeert meerdere SQL-instructies die samen moeten slagen of samen moeten falen. Je zet setAutoCommit op false, zodat jij beslist wanneer er bevestigd wordt. Lukt alles, dan maak je de wijzigingen definitief met commit; loopt er iets mis, dan draai je alles terug met rollback. Zolang er geen commit is gebeurd, zien andere gebruikers de wijzigingen niet. Zo houd je de databank in een consistente toestand, ook bij een fout halverwege.

Een transactie is een reeks SQL-instructies die in zijn geheel moet lukken. Mislukt er één, dan doe je een rollback en sta je terug in de begintoestand. Lukt alles, dan doe je een commit en wordt de wijziging zichtbaar voor alle gebruikers.

TermUitleg
commitTransactie geslaagd, wijzigingen definitief opslaan
rollbackFout opgetreden, alles ongedaan maken
autocommitStandaard AAN: na elke SQL automatisch een commit

Autocommit uitschakelen en zelf committen

Bij transacties zet je eerst setAutoCommit(false), zodat jij beslist wanneer er een commit() of rollback() gebeurt.

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.Statement;

public class Main {
    public static void main(String[] args) {
        Connection connectie = null;
        Statement statement = null;

        try {
            connectie = DriverManager.getConnection
                    ("jdbc:mysql://localhost/demo",
                            "root", "");
            statement = connectie.createStatement();
            connectie.setAutoCommit(false);
            statement.executeUpdate
                    ("DELETE FROM employees WHERE department='HR'");
            statement.executeUpdate
                    ("UPDATE employees SET salary=300000 " +
                            "WHERE department='Engineering'");
            boolean transactieGoedgekeurd = true;
            if (transactieGoedgekeurd) {
                connectie.commit();
            } else {
                connectie.rollback();
            }
        } catch (Exception exceptie) {
            exceptie.printStackTrace();
        } finally {
            try {
                statement.close();
                connectie.close();
            } catch (Exception exceptie) {
                exceptie.printStackTrace();
            }
        }
    }
}
⚠️Zolang er geen commit() gedaan is, zijn de wijzigingen niet zichtbaar voor andere gebruikers van de databank.

JDBC: ResultSet-metadata

 ·  ResultSetMetaData

ResultSetMetaData beschrijft de kolommen van het resultaat van een query, niet de waarden zelf. Je haalt het op met resultaat.getMetaData en vraagt met getColumnCount hoeveel kolommen er zijn. Daarna loop je over de kolommen, geteld vanaf 1, en vraag je per kolom de naam, het type en of die automatisch oploopt. Zo kan je een resultaat verwerken zonder vooraf de structuur te kennen. Dit is vooral nuttig voor algemene tools die met om het even welke tabel moeten werken.

Waar DatabaseMetaData over de hele databank gaat, beschrijft ResultSetMetaData net het resultaat van één query. Je vraagt het op met resultaat.getMetaData() en komt zo te weten hoeveel kolommen er zijn en hoe ze heten. Per kolom kun je ook het type opvragen of nakijken of ze automatisch oploopt. Je loopt over de kolommen met een lus die telt vanaf 1, want de nummering is one-based. Zo schrijf je code die met elk resultaat overweg kan, zonder de kolommen op voorhand te kennen.

Exceptions (try / catch / finally)

 ·  fouten opvangen zonder dat het programma crasht

Een exception is een fout die optreedt tijdens het uitvoeren, bijvoorbeeld delen door nul of een bestand dat niet bestaat. Code die kan mislukken zet je in een try-blok; bij een fout springt Java naar het bijbehorende catch-blok in plaats van te crashen. In catch vang je een specifiek fouttype op en bepaal je wat er dan moet gebeuren. Het finally-blok loopt altijd, ook na een fout, en gebruik je om op te kuisen, zoals verbindingen sluiten. Zo houd je je programma onder controle wanneer er iets misgaat.

Code die kan mislukken (een berekening, een bestand, een databank) zet je in een try. Loopt het mis, dan springt Java naar het catch-blok. Het finally-blok loopt altijd, ook bij een fout, en gebruik je om op te kuisen (bijvoorbeeld .close()).

public class Main {
    public static void main(String[] args) {
        int deler = 0;

        try {
            int resultaat = 10 / deler;          // kan een fout geven
            System.out.println(resultaat);
        } catch (ArithmeticException exceptie) {  // specifieke fout opvangen
            System.out.println("Delen door nul!");
            exceptie.printStackTrace();
        } finally {
            System.out.println("Dit blok loopt ALTIJD.");
        }
    }
}

Een String is een stuk tekst, en Java geeft je een hoop kant-en-klare methoden om ermee te werken. Je vraagt de lengte op, haalt er stukken uit, zoekt erin of vervangt delen. Math & getallen

 ·  java.lang.Math

De klasse Math bundelt veelgebruikte wiskundige bewerkingen die je rechtstreeks kan oproepen. Je vindt er onder andere abs voor de absolute waarde, max en min, pow voor machten en sqrt voor de vierkantswortel. Met round, floor en ceil rond je af naar het dichtstbijzijnde, naar beneden of naar boven. Math.random geeft een willekeurig kommagetal tussen 0 en net niet 1, dat je kan omrekenen naar een bereik. Alle methoden zijn static, dus je gebruikt ze via Math gevolgd door de naam, zonder een object te maken.

De klasse Math bundelt veelgebruikte rekenfuncties die altijd beschikbaar zijn. Je vindt er onder andere de absolute waarde, de grootste of kleinste van twee getallen, machten en wortels. Voor afronden heb je round, floor en ceil, die elk anders met de komma omgaan. Met Math.random krijg je een willekeurig kommagetal tussen 0 en net niet 1.
